# .claude/ — AI Development Configuration

This directory contains configuration for AI coding tools — primarily [Claude Code](https://docs.anthropic.com/en/docs/claude-code), with cross-tool support via `AGENTS.md`.

## Prerequisites

### Codex CLI (Required for audit commands)

The `codex-toolkit` plugin (`codex-toolkit@xiaolai`) uses OpenAI's Codex as an independent second opinion for code audits. The plugin is installed as a Claude Code plugin and provides its own MCP server — no manual `.mcp.json` entry needed.

Install Codex globally and log in with your subscription:

```bash
npm install -g @openai/codex
codex login                   # Log in with your ChatGPT subscription (recommended)
```

Verify it's on your PATH:

```bash
codex --version
```

> **Prefer subscription auth over API keys.** `codex login` uses your ChatGPT Plus ($20/mo) or Pro ($200/mo) subscription, which is dramatically cheaper than `OPENAI_API_KEY` pay-per-token billing for heavy coding sessions. API keys work as a fallback — see `codex login --with-api-key`.

### Why Codex?

Using a second AI model for code review catches blind spots that a single model might miss. Claude writes the code; Codex audits it independently. This cross-model verification is built into the `/codex-toolkit:audit` and `/fix-issue` workflows.

## How Codex MCP Works

The `codex-toolkit@xiaolai` plugin bundles its own `.mcp.json` that registers Codex as an MCP tool (`mcp__plugin_codex-toolkit_codex__codex`). Claude Code auto-loads this when the plugin is enabled. Codex runs in a sandboxed read-only context — it can read the codebase but cannot modify files.

Project-specific Codex settings live in `.codex-toolkit.md` at the project root (generated by `/codex-toolkit:init`).

## Directory Structure


### Settings Files

| File | Shared? | Purpose |
|------|---------|---------|
| `settings.json` | Yes | Team-wide plugin config (e.g., enabled plugins) |
| `settings.local.json` | **No** (gitignored) | Personal permissions, tool approvals, output style |

`settings.local.json` accumulates permissions as you approve tool calls during sessions. It's machine-specific and should not be committed.

### Hooks (`hooks/`)

Hooks run automatically at specific points in the Claude Code lifecycle:

| File | Trigger | Purpose |
|------|---------|---------|
| `refine_prompt.sh` | UserPromptSubmit | When a prompt starts with `::` or `>>`, sends it for translation/refinement, copies the result to clipboard, and blocks the original. |

**`refine_prompt.sh`** — Opt-in via `::` or `>>` prefix. Translates non-English prompts to English and optimizes prompt structure for better AI coding results. See the [Prompt Refinement guide](https://vmark.app/guide/users-as-developers/prompt-refinement) for details.

## Bot Cost Reports

Each Claude bot workflow (review, audit, fix) generates a cost report after every run, visible in the Actions run's **Step Summary** tab. The report includes workflow name, event type, model, duration, and status.

Composite action: `.github/actions/claude-report/action.yml`
