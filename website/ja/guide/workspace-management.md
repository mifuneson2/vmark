# ワークスペース管理

VMark のワークスペースとは、プロジェクトのルートとして開かれたフォルダーです。ワークスペースを開くと、サイドバーにファイルツリーが表示され、クイックオープンがすべての Markdown ファイルをインデックス化し、ターミナルがプロジェクトルートで起動し、開いているタブが次回のために記憶されます。

ワークスペースなしでも個別のファイルを開けますが、ファイルエクスプローラー、プロジェクト内検索、セッションの復元が使えなくなります。

## ワークスペースを開く

| 方法 | 手順 |
|-----|-----|
| メニュー | **ファイル > ワークスペースを開く** |
| クイックオープン | `Mod + O`、次に下部の**参照... **を選択 |
| ドラッグ&ドロップ | Finder から Markdown ファイルをウィンドウにドラッグ — VMark がプロジェクトルートを検出してワークスペースを自動的に開く |
| 最近使ったワークスペース | **ファイル > 最近使ったワークスペース**から以前のプロジェクトを選択 |

ワークスペースを開くと、VMark はファイルエクスプローラー付きのサイドバーを表示します。ワークスペースが以前に開かれていた場合、以前に開いていたタブが復元されます。

::: tip
現在のウィンドウに未保存の変更がある場合、VMark は作業を置き換える代わりに新しいウィンドウでワークスペースを開くことを提案します。
:::

## ファイルエクスプローラー

ファイルエクスプローラーはワークスペースが開いているときにサイドバーに表示されます。ワークスペースフォルダーをルートとした Markdown ファイルのツリーを表示します。

### ナビゲーション

- フォルダーを**シングルクリック**して展開または折りたたむ
- ファイルを**ダブルクリック**または**Enter**でタブで開く
- 非 Markdown ファイルはシステムのデフォルトアプリケーションで Open

### ファイル操作

任意のファイルまたはフォルダーを右クリックしてコンテキストメニューにアクセスします:

| アクション | 説明 |
|---------|-----|
| 開く | ファイルを新しいタブで開く |
| 名前変更 | ファイルまたはフォルダーの名前をインラインで編集（`F2`でも可） |
| 複製 | ファイルのコピーを作成 |
| 移動先... | ダイアログ経由でファイルを別のフォルダーに移動 |
| 削除 | ファイルまたはフォルダーをシステムのゴミ箱に移動 |
| パスをコピー | 絶対ファイルパスをクリップボードにコピー |
| Finder で表示 | Finder でファイルを表示（macOS） |
| 新規ファイル | この場所に新しい Markdown ファイルを作成 |
| 新規フォルダー | この場所に新しいフォルダーを作成 |

ツリー内で直接**ドラッグ&ドロップ**でフォルダー間でファイルを移動できます。

### 表示切り替え

デフォルトでエクスプローラーは Markdown ファイルのみを表示し、ドットファイルを非表示にします。2 つの切り替えでこれを変更できます:

| 切り替え | ショートカット | 動作 |
|--------|------------|-----|
| 隠しファイルを表示 | `Mod + Shift + .`（macOS） / `Ctrl + H`（Win/Linux） | ドットファイルと隠しフォルダーを表示 |
| すべてのファイルを表示 | _（設定またはコンテキストメニュー）_ | ドキュメントと並んで非 Markdown ファイルを表示 |

両方の設定はワークスペースごとに保存され、セッション間で持続します。

### 除外フォルダー

特定のフォルダーはデフォルトでツリーから除外されます:

- `.git`
- `node_modules`

これらのデフォルトはワークスペースが最初に開かれたときに適用されます。

## クイックオープン

`Mod + O`を押してクイックオープンオーバーレイを開きます。3 つのソースにわたるファジー検索を提供します:

1. 以前に開いた**最近のファイル**
2. 現在のウィンドウの**開いているタブ**（ドットインジケーターでマーク）
3. ワークスペース内の**すべての Markdown ファイル**

いくつかの文字を入力してフィルタリングします — マッチングはファジーなので、`rme`で`README.md`が見つかります。矢印キーでナビゲートして**Enter**で開きます。下部に固定された**参照... **行でファイルダイアログが開きます。

| アクション | ショートカット |
|---------|-------------|
| クイックオープンを開く | `Mod + O` |
| 結果をナビゲート | `Up / Down` |
| 選択したファイルを開く | `Enter` |
| 閉じる | `Escape` |

::: tip
ワークスペースなしでも、クイックオープンは動作します — 最近のファイルと開いているタブは表示されますが、ファイルツリーを検索できません。
:::

## 最近使ったワークスペース

VMark は最近開いた最大 10 のワークスペースを記憶します。メニューバーの**ファイル > 最近使ったワークスペース**からアクセスします。

- ワークスペースは最後に開いた時刻順（最新順）でソートされます
- リストはすべての変更でネイティブメニューに同期されます
- **最近使ったワークスペースをクリア**を選択してリストをリセット

## ワークスペース設定

各ワークスペースはセッション間で持続する独自の設定を持ちます。設定は VMark のアプリケーションデータディレクトリに保存されます — プロジェクトフォルダー内ではありません — のでワークスペースはクリーンに保たれます。

以下の設定がワークスペースごとに保存されます:

| 設定 | 説明 |
|-----|-----|
| 除外フォルダー | ファイルエクスプローラーから非表示にされたフォルダー |
| 隠しファイルを表示 | ドットファイルが表示されているかどうか |
| すべてのファイルを表示 | 非 Markdown ファイルが表示されているかどうか |
| 最後に開いていたタブ | 次回起動時のセッション復元用ファイルパス |

::: tip
ワークスペースの設定はフォルダーパスに関連付けられています。同じマシンで同じフォルダーを開くと、別のウィンドウからでも常に設定が復元されます。
:::

## セッションの復元

ワークスペースが開いているウィンドウを閉じると、VMark は開いているタブのリストをワークスペースの設定に保存します。次回同じワークスペースを開くと、これらのタブが自動的に復元されます。

- 保存済みファイルパスのあるタブのみが復元されます（無題のタブは保持されません）
- 最後のセッション以降にファイルが移動または削除された場合、静かにスキップされます
- セッションデータはウィンドウの閉じ時とワークスペースの閉じ時（**ファイル > ワークスペースを閉じる**）に保存されます

## マルチウィンドウ

各 VMark ウィンドウは独自の独立したワークスペースを持てます。これにより複数のプロジェクトを同時に操作できます。

- **ファイル > 新規ウィンドウ**で新しいウィンドウが開きます
- 新しいウィンドウでワークスペースを開いても他のウィンドウに影響しません
- ウィンドウのサイズと位置はウィンドウごとに記憶されます

Finder から Markdown ファイルをドラッグし、現在のウィンドウに未保存の作業がある場合、VMark は自動的にファイルのプロジェクトを新しいウィンドウで開きます。

## ターミナル統合

統合ターミナルは作業ディレクトリとしてワークスペースルートを自動的に使用します。ワークスペースを開くまたは切り替えると、すべてのターミナルセッションが新しいルートに`cd`します。

`VMARK_WORKSPACE`環境変数はすべてのターミナルセッションでワークスペースパスに設定されるため、スクリプトからプロジェクトルートを参照できます。
