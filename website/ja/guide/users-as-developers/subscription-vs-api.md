# サブスクリプション vs API プライシング

AI コーディングツールには 2 つの認証方法があります: **サブスクリプションプラン**と**API キー**。継続的なコーディングセッション（バイブコーディング）では、サブスクリプションが劇的に安くなります — 同じ作業で API の請求よりも 10〜30 倍安くなることがよくあります。[^1]

## コストの違い

典型的なアクティブなコーディングセッションでは、1 時間に数十万トークンを使用します。コストを比較するとこうなります:

### Claude Code

| 方法 | コスト | 得られるもの |
|--------|------|-------------|
| **Claude Max**（サブスクリプション） | $100〜200/月 | コーディングセッション中の無制限使用 |
| **API キー**（`ANTHROPIC_API_KEY`） | $600〜2,000+/月 | トークンごとに支払い; 重い使用はすぐに積み上がる |

**認証コマンド:**
```bash
claude          # Claude Maxサブスクリプションで自動ログイン（推奨）
```

### Codex CLI (OpenAI)

| 方法 | コスト | 得られるもの |
|--------|------|-------------|
| **ChatGPT Plus**（サブスクリプション） | $20/月 | 中程度の使用 |
| **ChatGPT Pro**（サブスクリプション） | $200/月 | ヘビーユース |
| **API キー**（`OPENAI_API_KEY`） | $200〜1,000+/月 | トークンごとに支払い |

**認証コマンド:**
```bash
codex login     # ChatGPTサブスクリプションでログイン（推奨）
```

### Gemini CLI (Google)

| 方法 | コスト | 得られるもの |
|--------|------|-------------|
| **無料枠** | $0 | 寛大な無料クォータ |
| **Google One AI Premium** | 約 $20/月 | より高い制限 |
| **API キー**（`GEMINI_API_KEY`） | 変動 | トークンごとに支払い |

**認証コマンド:**
```bash
gemini          # Googleアカウントでログイン（推奨）
```

## 経験則

> **サブスクリプション = 継続的なコーディングセッションで 10〜30 倍安い**

計算は単純です: サブスクリプションは月額固定料金を提供し、API 請求はトークンごとに課金します。AI コーディングツールは非常にトークンを消費します — ファイル全体を読み込み、長いコードブロックを生成し、複数ラウンドの編集を繰り返します。単一の複雑な機能で数百万トークンを消費することがあります。[^2]

## API キーがまだ意味をなす場合

API キーが正しい選択となる場合:

| ユースケース | 理由 |
|----------|-----|
| **CI/CD パイプライン** | 短時間で頻度の低い自動化ジョブ |
| **軽いまたは偶発的な使用** | 週に数回のクエリ |
| **プログラマティックアクセス** | API を直接呼び出すスクリプトと統合 |
| **チーム/組織の請求** | API 使用ダッシュボードを通じた集中請求 |

インタラクティブなコーディングセッション — AI と何時間もやり取りする場合 — では、サブスクリプションが常にコスト面で勝ります。[^3]

## VMark での設定

VMark の`AGENTS.md`はプロジェクトの規約としてサブスクリプション優先の認証を強制しています。リポジトリをクローンして AI コーディングツールを開くと、サブスクリプション認証を使うよう通知されます:

```
Prefer subscription auth over API keys for all AI coding tools.
```

認証後、3 つのツールはすべてそのまま動作します:

```bash
# 推奨: サブスクリプション認証
claude              # Claude CodeとClaude Max
codex login         # Codex CLIとChatGPT Plus/Pro
gemini              # Gemini CLIとGoogleアカウント

# フォールバック: APIキー
export ANTHROPIC_API_KEY=sk-...
export OPENAI_API_KEY=sk-...
export GEMINI_API_KEY=AI...
```

::: tip macOS GUI アプリの PATH
macOS GUI アプリ（Spotlight から起動したターミナルなど）は最小限の PATH を持ちます。ツールがターミナルで動作するのに Claude Code が見つけられない場合、バイナリの場所がシェルプロフィール（`~/.zshrc`または`~/.bashrc`）にあることを確認してください。
:::

[^1]: 典型的な集中的な AI コーディングセッションでは、1 回のインタラクションで 50,000〜100,000 以上のトークンを消費します。現在の API レート（例: Claude Sonnet は入力/出力それぞれ 100 万トークンあたり$3/$15）で、ヘビーユーザーは月額 API 費用$200〜$2,000+を報告しています — サブスクリプションプランは無制限使用で$100〜$200/月が上限です。この格差は使用強度とともに増大します: 軽いユーザーはどちらでも同様のコストになる可能性がありますが、継続的なバイブコーディングセッションはサブスクリプションを明らかな勝者にします。参照: [AI Development Tools Pricing Analysis](https://vladimirsiedykh.com/blog/ai-development-tools-pricing-analysis-claude-copilot-cursor-comparison-2025) (2025); [Claude Code Token Limits Guide](https://www.faros.ai/blog/claude-code-token-limits), Faros AI (2025).

[^2]: AI コーディングエージェントは単純なチャット操作よりはるかに多くのトークンを消費します。なぜなら、ファイル全体をコンテキストに読み込み、マルチファイル編集を生成し、反復的な修正テストループを実行し、長いセッションにわたって会話履歴を維持するからです。単一の複雑な機能実装は、それぞれ数千トークンを消費する数十のツール呼び出しを含む場合があります。コンテキストウィンドウ自体がコストドライバーになります — より大きなウィンドウはより良い結果を可能にしますが、トークン使用量が増加します。参照: [The Real Cost of Vibe Coding](https://smarterarticles.co.uk/the-real-cost-of-vibe-coding-when-ai-over-delivers-on-your-dime) (2025).

[^3]: より広い SaaS 業界は、フラットサブスクリプションと使用量ベースのコンポーネントを組み合わせたハイブリッドプライシングモデルに移行しています。2023 年までに、SaaS ビジネスの 46% が使用量ベースのプライシングを採用し、それを使用する企業は 137% のネットドル維持率を報告しています。しかし、すべてのクエリが顕著なコンピューティングを消費する AI 搭載ツールでは、純粋な使用量ベースのプライシングはユーザーを予測不能なコストにさらします — これがフラットレートサブスクリプションがヘビーな個人ユーザーにとって魅力的であり続ける理由です。参照: [The State of SaaS Pricing Strategy](https://www.invespcro.com/blog/saas-pricing/) (2025); [The Evolution of Pricing Models for SaaS Companies](https://medium.com/bcgontech/the-evolution-of-pricing-models-for-saas-companies-6d017101d733), BCG (2024).
