# Website Internationalization Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add multi-language support to the VMark VitePress documentation website — 10 languages, 30 pages, subdirectory URL pattern.

**Architecture:** VitePress built-in `locales` config with per-locale config files. English at root (`/guide/`), other languages at `/{lang}/guide/`. AI-translated content via the existing `scripts/translate.ts` adapted for markdown. Language switcher auto-rendered by VitePress.

**Tech Stack:** VitePress i18n locales, Claude API for translation.

**Spec:** `docs/superpowers/specs/2026-03-16-internationalization-design.md` (Section 4)

## Chunk 2: Content Translation

### Task 4: Create markdown translation script

**Files:**
- Create: `scripts/translate-docs.ts`

A script specifically for translating markdown documentation pages. Different from the app's `translate.ts` because:
- Input/output are markdown files (not JSON/YAML)
- Must preserve frontmatter, code blocks, HTML tags, and file paths
- Must rewrite internal links to locale-prefixed paths (`/guide/features` → `/{lang}/guide/features`)
- External URLs stay unchanged

- [ ] **Step 1: Create the script**

```typescript
#!/usr/bin/env -S node --import tsx
/**
 * Translate VitePress markdown documentation pages.
 *
 * Usage:
 *   tsx scripts/translate-docs.ts --lang zh-CN
 *   tsx scripts/translate-docs.ts --lang ja --file guide/features.md
 *   tsx scripts/translate-docs.ts --lang all
 */
```

Key features:
- Reads all `.md` files from `website/` root (excluding node_modules, .vitepress)
- For each file, sends to Claude with instructions to:
  - Translate prose only (not code blocks, not frontmatter values, not URLs)
  - Preserve markdown syntax exactly
  - Keep `:::tip`, `:::warning` container markers
  - Keep image paths and HTML tags unchanged
  - Rewrite internal links: `/guide/X` → `/{lang}/guide/X`, `/download` → `/{lang}/download`
- Writes translated file to `website/{lang}/` mirroring the directory structure
- Supports `--lang all` to translate all 9 languages
- Supports `--file` to translate a single file
- Diff-aware: skip files that haven't changed

- [ ] **Step 2: Test with one file**

```bash
tsx scripts/translate-docs.ts --lang zh-CN --file guide/index.md --dry-run
```

- [ ] **Step 3: Commit**

```bash

git commit -m "feat(i18n): add markdown documentation translation script"
```
